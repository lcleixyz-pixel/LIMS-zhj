# Jewelry QMS 使用与体系文件适配指南

面向珠宝检测实验室质量负责人，说明如何将现有纸质、Word、Excel 体系资料纳入当前 ThinkPHP 8 版 `jewelry-qms`。

## 1. 文件层级与审批

| 层级 | 代码前缀示例 | 审批 |
|------|--------------|------|
| 质量手册 | QM- | 编制 -> 审核 -> 批准 |
| 程序文件 | QP- | 编制 -> 审核 -> 批准 |
| 作业指导书 | SOP- | 编制 -> 批准 |
| 记录表格 | REC- | 编制 -> 批准 |

系统配置位置：`jewelry-qms/config/qms.php` 中的 `docLevels` 与 `approvalRules`。

## 2. 适配现有 Word 模板

### 步骤 A：上传模板

1. 登录 QMS。
2. 进入 **文件控制** -> **文件模板管理**。
3. 新增模板，选择适用层级，上传 `.docx` 文件。
4. 建议每一类文件至少准备 1 个标准模板，包含页眉页脚、文件编号栏、修订栏等。

### 步骤 B：新建受控文件

1. 进入 **文件控制** -> **新建文件**。
2. 填写文件编号、文件名称、层级、归口部门、版本等信息。
3. 上传当前有效版 Word/PDF，或先选择模板后再上传编辑后的文件。
4. 指定审核人、批准人。质量手册和程序文件需要审核、批准两级人员；作业指导书和记录表格只需批准人。

### 步骤 C：发布与修订

- 草稿提交审核后，审批人登录系统处理待办。
- 全部审批通过后，文件状态进入发布链路。
- 需要换版时，通过修订入口填写变更原因并上传新版文件，重新走审批。

## 3. 批量导入现有台账

当前导入入口支持 CSV。若已有 Excel 台账，请先另存为 CSV 后上传。

文件台账建议字段：

| 列名 | 对应字段 |
|------|----------|
| 文件编号 | `doc_number` |
| 文件名称 | `title` |
| 层级 | `level`，取值 1-4 |
| 版本 | `version` |
| 归口部门 | 部门名称，需先在系统建部门 |
| 生效日期 | `effective_date` |

系统还支持设备、人员基础数据的 CSV 导入，用于快速建立 QMS 初始台账。

## 4. 当前模块与 17025 条款对应

| 模块 | 主要条款 | 当前状态 |
|------|----------|----------|
| 文件控制 | 8.2、8.3 | 四层级、模板、审批、发布、修订 |
| 内部审核 | 8.8 | 计划、日程、检查表、发现，发现可触发 CAPA |
| 管理评审 | 8.9 | 输入汇总、会议记录、决议事项跟踪 |
| CAPA | 8.7 | 来源关联、原因分析、措施实施、验证关闭 |
| 设备与校准 | 6.4、6.5 | 设备台账、校准记录、到期提醒 |
| 培训与能力 | 6.2 | 培训记录、完成标记、能力确认 |
| 供应商 | 6.6 | 评价、状态驱动、合格名录 |
| 客户投诉 | 7.9 | 受理、调查、处置、关闭、CAPA 关联 |
| 不符合工作 | 7.10 | 评估、纠正、验证、关闭、CAPA 关联 |

当前版本已完成 P1 业务深化，不再只是 CRUD 骨架。上线前仍建议按实验室真实记录表逐项核对字段、状态和审批责任。

## 5. 体系策划中心

体系策划中心用于把外部标准、质量手册、程序文件、记录表格和运行模块串成可追溯骨架。

推荐工作顺序：

1. 在 **外部依据** 登记或上传 `GB/T 27025`、`CNAS`、`CMA`、`A015` 等文件。
2. 维护查新日期、查新结论、证据来源和下次查新日期。
3. 抽取正式条款，形成条款库。
4. 基于 `27025` 初始化无编号要素，并把补充条款映射到已有要素。
5. 将现用质量手册、程序文件和记录表格结构化为 Markdown 内容块。
6. 人工复核块级追溯、记录表格 schema 和智能体建议。
7. 渲染系统包，形成可归档的完整体系文件输出。

重要边界：

- 要素不显示 `6.2`、`7.8` 等编号，编号只作为条款或手册章节的关联信息出现。
- 手册章节编号属于质量手册章节，不属于要素。
- 记录表格 schema 必须来自程序文件要求和现用表格原件，不能只按表名猜测。
- 智能体建议只作为待复核清单，正式数据仍由人工确认。

## 6. 用户与权限

角色配置位于 `jewelry-qms/config/qms.php`：

| 角色 | 建议用途 |
|------|----------|
| `admin` | 系统管理员，维护基础配置和账号 |
| `quality_manager` | 质量负责人，处理质量体系全局事项 |
| `auditor` | 内审员，参与内审计划、检查和发现处理 |
| `department_head` | 部门负责人，维护本部门相关记录 |
| `staff` | 一般人员，查看文件和提交投诉等有限操作 |

审批链中被指定的用户需要在用户信息中具备审批人属性，并拥有相应模块访问权限。

## 7. 与检测业务系统并存

- QMS 不替代委托、样品、检测、报告、收费等 LIMS 功能。
- 投诉、不符合等记录可手工填写关联报告编号，便于追溯。
- 人员、设备、客户、报告编号等主数据宜由 LIMS 通过 API 或只读视图同步，避免双份维护。

## 8. 常见问题

**Q：上传 Word 后能否在线改？**  
A：当前版本是上传/下载模式。在线编辑可在后续版本集成 ONLYOFFICE。

**Q：记录表格是否必须三级审批？**  
A：否。当前配置为记录表格两级审批：编制 -> 批准，可在 `config/qms.php` 调整。

**Q：Excel 能否直接导入？**
A：当前界面允许选择 Excel 文件，但实际导入逻辑要求先另存为 CSV。建议统一使用 CSV。

**Q：废止文件如何处理？**  
A：数据库已支持 `obsolete` 状态，实际使用时应结合修订记录保留历史版本，不建议物理删除。

**Q：为什么体系要素没有编号？**

A：编号属于外部条款或手册章节。要素是“人员”“设备”“记录控制”等体系概念节点，系统用不可见 `key/id` 关联，界面只显示中文名称。

**Q：现有 Word、Excel 是否要全部丢掉？**

A：不需要。原文件应固定归档，抽取为 Markdown 和 schema 后进入系统维护。后续修改以结构化内容为主，同时保留渲染输出和历史归档。
