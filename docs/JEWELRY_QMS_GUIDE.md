# Jewelry QMS 使用与体系文件适配指南

面向珠宝检测实验室质量负责人，说明如何将**现有纸质/Word 体系文件**纳入系统。

## 1. 文件层级与审批（与贵实验室约定一致）

| 层级 | 代码前缀示例 | 审批 |
|------|--------------|------|
| 质量手册 | QM- | 编制 → 审核 → 批准 |
| 程序文件 | QP- | 编制 → 审核 → 批准 |
| 作业指导书 | SOP- | 编制 → 批准 |
| 记录表格 | REC- | 编制 → 批准 |

系统配置位置：`jewelry-qms/app/Config/core.php` 中 `QMS.docLevels`、`QMS.approvalRules`。

## 2. 适配现有 Word 模板

### 步骤 A：上传模板

1. 登录 QMS → **文件控制** → **文件模板管理**
2. **新增** → 选择适用层级 → 上传 `.docx`
3. 为每一类文件至少准备 1 个标准模板（页眉页脚、文件编号栏、修订栏等）

### 步骤 B：新建受控文件

1. **文件控制** → **新建文件**
2. 填写与现有体系一致的**文件编号**、**名称**、**归口部门**
3. 上传当前有效版 Word，或先选模板再本地编辑后上传
4. 指定审核人、批准人（程序文件/手册需两级审批人）

### 步骤 C：发布与修订

- 草稿 → **提交审核** → 审批人登录后在文件详情页批准
- 全部通过后状态为 **已发布**
- 需换版时：**发起修订**，填写变更原因，上传新版 Word，重新走审批

## 3. 批量导入现有文件台账（建议）

若已有 Excel 总台账（约 50 份以内），建议字段：

| 列名 | 对应字段 |
|------|----------|
| 文件编号 | doc_number |
| 文件名称 | title |
| 层级 | level（1–4） |
| 版本 | version |
| 归口部门 | department_id（需先在系统建部门） |
| 生效日期 | effective_date |

可委托开发编写 `INSERT` 脚本或导入功能（P1 任务）。

## 4. 各模块与 17025 条款对应

| 模块 | 主要条款 |
|------|----------|
| 文件控制 | 8.2、8.3 |
| 内部审核 | 8.8 |
| 管理评审 | 8.9 |
| CAPA | 8.7 |
| 设备与校准 | 6.4、6.5 |
| 培训与能力 | 6.2 |
| 供应商 | 6.6 |
| 客户投诉 | 7.9 |
| 不符合工作 | 7.10 |

当前除**文件控制**外，其余模块为 CRUD 骨架，录入字段可按贵实验室记录表格逐项对齐。

## 5. 用户与权限

| 角色 | 建议 |
|------|------|
| admin | 系统管理员 1 人 |
| quality_manager | 质量负责人，全部审批 |
| auditor | 内审员 |
| department_head | 部门负责人，本部门文件编制 |
| staff | 一般查看 |

在用户管理中勾选 **审批人**（`is_approver`）方可进入审批链。

## 6. 与检测系统并存

- QMS **不替代** 报告编制与发放
- 投诉、不符合可手工填写**关联报告编号** `report_number`，便于追溯
- 人员、设备主数据宜与 LIMS 定期同步，避免双份维护（见 [ARCHITECTURE.md](ARCHITECTURE.md)）

## 7. 常见问题

**Q：上传 Word 后能否在线改？**  
A：当前为上传/下载模式；在线编辑需后续集成 ONLYOFFICE（参考 On-Premise 项目）。

**Q：记录表格是否必须三级审批？**  
A：否，记录表格为两级（编制→批准），可在配置中调整。

**Q：废止文件如何处理？**  
A：将状态改为 obsolete（后续版本可加「废止」按钮）；保留修订历史不物理删除。
