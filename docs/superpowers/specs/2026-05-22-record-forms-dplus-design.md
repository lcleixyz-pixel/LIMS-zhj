# 记录表格 D+ 方案设计

日期：2026-05-22

## 背景

现有记录表格主要来自 `现用文件/记录表格/记录表格2017`，其中大量文件是旧版 Word 表格。前一轮导入预览已经把空白模板、历史记录和人工确认项分开。本设计用于确认后续系统能力方向：不以逐字高保真复刻旧 Word 为主，而是将记录表格重构为系统原生可填写表单，并保留旧文件作为受控参考原件。

## 目标

- 支持用户在系统中编辑指定字段。
- 根据结构化字段生成可打印 PDF。
- 支持预览原始模板文件和系统重构后的打印模板。
- 保留原编号、原始附件、来源路径和受控状态。
- 让记录表格后续可查询、可复用、可校验、可版本管理。

## 非目标

- 第一阶段不实现在线 Word 编辑。
- 第一阶段不要求所有旧 Word 模板逐字逐页复刻。
- 第一阶段不直接把历史已填记录转为系统表单实例。
- 第一阶段不批量导入数据库，先完成样板和规格确认。

## 推荐路线

采用 D+ 方案：

1. 原始模板层：保存旧 Word/PDF/Excel 文件，作为受控附件和迁移依据。
2. 结构化表单层：为每个记录表格定义字段 schema，用户只编辑字段值。
3. 打印模板层：用 HTML/CSS 重构 A4 打印版式。
4. PDF 生成层：优先使用浏览器打印引擎或 wkhtmltopdf 生成 PDF，保留 mPDF 作为轻量备选。
5. 预览层：同时提供原始附件预览和系统打印预览。

## 功能设计

### 模板管理

模板管理页面显示记录表格编号、名称、模块、状态、原始附件、打印模板、字段数量和最近更新时间。模板详情页提供两个预览入口：

- 原始附件预览：展示迁移前 Word/PDF 文件。
- 打印模板预览：展示系统 HTML/CSS 渲染后的 A4 页面。

### 字段配置

每张表格维护一份字段 schema。字段配置先以 JSON 编辑和开发内置方式落地，样板稳定后再考虑做可视化字段配置器。

字段配置必须支持：

- 必填校验。
- 默认值。
- 下拉选项。
- 日期格式。
- 重复行表格。
- 打印绑定位置。

### 表格填写

用户从模板发起一条记录实例。系统根据字段 schema 渲染填写界面，保存草稿后可生成打印预览。生成 PDF 前必须经过字段校验。

实例状态建议：

- `draft`：草稿。
- `generated`：已生成 PDF。
- `locked`：已确认归档，不允许普通编辑。
- `voided`：作废。

### PDF 生成

第一阶段采用 HTML/CSS 打印模板作为唯一打印源。打印样式按 A4 页面设计，控制页边距、表格边框、分页、页眉页脚和签名区域。

生成引擎优先级：

1. Chromium/Playwright PDF：版式能力好，适合复杂 CSS。
2. wkhtmltopdf：参考项目已有类似经验，部署相对成熟。
3. mPDF：纯 PHP 部署轻，但复杂布局能力有限，仅作为简单表单备选。

### 附件和受控状态

旧 Word 文件不丢弃，继续作为受控附件保存。系统生成的 PDF 是新的记录产物，不覆盖原始模板。模板状态仍遵循既定策略，导入后的模板可设为 `published`，实例按业务流程进入草稿、生成或归档状态。

## 样板范围

第一阶段建议先做 5 类样板：

1. 人员培训：年度人员培训计划表、人员培训记录表。
2. 设备管理：设备期间核查记录、设备维护/使用记录。
3. 内审：内审检查表、不符合项报告、整改跟踪。
4. 管理评审：管理评审输入资料、会议记录、签到。
5. 质量控制：质量控制计划、质控结果记录。

这些样板覆盖普通单页表、重复行表、签名表、跨模块记录、审核整改类记录，足够验证后续批量重构方式。

## 迁移策略

1. 先从导入预览清单中选择样板表格。
2. 将原始文件作为受控附件导入模板记录。
3. 人工定义字段 schema。
4. 编写对应 HTML 打印模板。
5. 用模拟数据生成 PDF。
6. 对照原始模板和质量体系要求进行人工确认。
7. 确认样板后，再扩展到同模块其他表格。

## 风险与控制

- 旧表格版式差异大：通过样板先定模板规范，再批量复制。
- HTML 打印分页可能不稳定：优先使用浏览器打印引擎，并为每张模板做 PDF 快照验证。
- 字段过度自由会影响记录一致性：用 schema 限制字段类型、必填项和选项。
- 旧文件与新模板可能不一致：模板详情同时展示原始附件和系统打印预览，保留人工复核入口。
- 历史记录误入模板库：继续沿用导入预览清单的导入/跳过/人工确认规则。

## 验收标准

- 用户可以打开一张记录表格模板，看到原始附件预览和系统打印预览。
- 用户可以填写字段并保存草稿。
- 系统可以根据字段生成 A4 PDF。
- PDF 中编号、名称、页眉、表格线、签名区和日期字段清晰可打印。
- 生成的记录实例不覆盖原始模板文件。
- 至少 5 类样板表格通过人工预览确认。

## 后续实施顺序

1. 完成样板表格选择。
2. 建立记录表格模板和实例的数据结构。
3. 实现字段 schema 渲染和保存。
4. 实现 HTML 打印模板渲染。
5. 接入 PDF 生成引擎。
6. 加入原始附件和系统打印双预览。
7. 用样板表格验证后进入批量重构。
